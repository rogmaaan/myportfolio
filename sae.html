GEST PLANTE (SAÉ S5/S6)

Java Node.js MongoDB Vue.js Arduino C++ Android, Kotlin

Présentation du projet GEST PLANTE

Le projet GEST PLANTE est une application complexe développée dans le cadre de la SAÉ S5/S6. Elle exploite l'IoT (ESP32, divers capteurs) afin de surveiller en temps réel les conditions environnementales des plantes domestiques. Les données collectées sont traitées, stockées dans MongoDB et visualisées via une interface intuitive développée en Vue.js. Ce projet inclut également un volet IA (analyse des images des plantes), prévu pour être finalisé au semestre S6.

Trace 1 : Architecture Logicielle et Matérielle du Projet GEST PLANTE

Explication générale du schéma :

Le schéma présenté ici illustre l’architecture complète et intégrée du projet GEST PLANTE, conçu dans le cadre de la SAÉ S5-S6. Ce projet vise à créer une solution intelligente de gestion de plantes domestiques, s’appuyant sur l’interconnexion d’éléments matériels (IoT), logiciels, et des techniques avancées d'analyse et de visualisation de données.

Analyse détaillée par composants :

  1. Microcontrôleur ESP32 avec capteurs Arduino :
    Le microcontrôleur ESP32 constitue la pierre angulaire matérielle du système. Associé à plusieurs capteurs (température/humidité DHT11, luminosité LDR, niveau d’eau, humidité du sol), il collecte en continu des données environnementales en temps réel.
    Technologie utilisée : Arduino IDE, ESP32.
    Rôle précis :
    Capturer les données des capteurs en temps réel.
    Assurer la transmission fiable des données via un protocole réseau (TCP/IP) au serveur Java central.
    Compétences illustrées : Développement d’applications IoT embarquées, gestion des capteurs, et programmation réseau.
  2. Serveur Central Java TCP/IP :
    Ce serveur Java joue un rôle central dans la gestion des données transmises depuis les microcontrôleurs ESP32. Il garantit la réception simultanée de multiples flux de données grâce à une architecture multithreadée efficace.
    Technologie utilisée : Java (Sockets TCP), IntelliJ IDEA.
    Rôle précis :
    Réception simultanée et robuste des flux de données.
    Traitement primaire (pré-analyse, validation des données).
    Acheminement sécurisé vers l’API REST pour stockage permanent.
    Compétences illustrées : Programmation réseau avancée, multithreading, gestion efficace des données.
  3. API REST en Node.js & Base de Données MongoDB :
    L’API REST conçue avec Node.js et Express est l’interface entre le serveur Java et la base de données MongoDB. Elle permet aussi l’exécution de scripts Python pour effectuer des analyses approfondies sur les données collectées.
    Technologie utilisée : Node.js, Express, MongoDB, Mongoose, Visual Studio Code.
    Rôle précis :
    Stocker de manière structurée les données issues des capteurs.
    Permettre une récupération rapide et flexible des données pour la visualisation frontend.
    Appeler et gérer des scripts Python pour des analyses de données complexes.
    Compétences illustrées : Développement backend, gestion de bases de données non relationnelles, création et utilisation d’API REST sécurisée.
  4. Serveur d'analyse Python :
    Un serveur Python dédié traite les images capturées par l'application mobile. Cette composante du système utilise des algorithmes d'analyse visuelle avancée pour extraire des informations pertinentes concernant la santé des plantes.
    Technologie utilisée : Python (TensorFlow/OpenCV), Visual Studio Code.
    Rôle précis :
    Réceptionner et analyser les images transmises par l’application mobile.
    Détection préliminaire d’anomalies (maladies des plantes, croissance, etc.).
    Transmission des résultats au serveur Java pour stockage via l'API REST.
    Compétences illustrées : Analyse multimédia, traitement d’images, intelligence artificielle basique.
  5. Application Mobile (Android, Kotlin) :
    L’application mobile, développée en Kotlin pour Android, permet aux utilisateurs de capturer facilement des images de leurs plantes et de les envoyer au serveur d'analyse Python.
    Technologie utilisée : Kotlin, Android Studio.
    Rôle précis :
    Capturer des images directement ou depuis la galerie.
    Assurer une communication fiable avec le serveur Python d'analyse.
    Compétences illustrées : Développement mobile natif, gestion des autorisations utilisateur, design d’interface intuitive.
  6. Frontend Vue.js :
    Le frontend, développé avec le framework Vue.js, présente de manière visuelle et interactive les données environnementales collectées, permettant aux utilisateurs de comprendre immédiatement l'état et les besoins de leurs plantes.
    Technologie utilisée : Vue.js, HTML, CSS, JavaScript.
    Rôle précis :
    Visualiser dynamiquement les données à travers des graphiques interactifs.
    Fournir une interface utilisateur claire, accessible et sécurisée.
    Compétences illustrées : Création d’interfaces dynamiques, gestion de données en temps réel, sécurité frontale (gestion d’authentification et autorisations).

Flux de Données et Intégration des Composants :

Ce schéma montre explicitement le parcours logique et physique des données à travers tout le système :

  1. Les données initiales viennent des capteurs IoT (ESP32).
  2. Elles sont transmises au serveur Java central via TCP.
  3. Les données traitées sont stockées via l’API REST dans la base MongoDB.
  4. L’application mobile envoie des images au serveur d'analyse Python.
  5. Python retourne des informations traitées au serveur central Java.
  6. Le frontend récupère les données via l'API pour une visualisation utilisateur claire et utile.

Trace 2: Serveur TCP Java multiclient

Savoir-faire visé

Développer une application réseau en Java capable de gérer plusieurs connexions clientes simultanées grâce au multithreading, dans le cadre de la centralisation des mesures IoT.

Contexte du projet

Dans notre projet GEST PLANTE, nous avions besoin d’un serveur capable de recevoir les mesures envoyées en temps réel par plusieurs capteurs (humidité, température, lumière…). Chaque capteur étant connecté à un microcontrôleur ESP32, nous avons développé un serveur TCP en Java utilisant des threads pour traiter plusieurs connexions simultanément, sans blocage.

Traces (captures commentées)

Extrait de code 

while(true) {
    sock = conn.accept(); // Attend une connexion
    ThreadServer t = new ThreadServer(idThread++, sock, exchanger); // Crée un thread client
    t.start(); // Lance le thread pour traitement parallèle
}

Ce code montre clairement la mise en place d’un serveur TCP multithreadé. Chaque connexion client est isolée dans un thread, ce qui garantit la stabilité du système, même si plusieurs capteurs envoient des données en même temps.

Ce que j’ai fait

  • Créé une classe MainServer qui écoute en continu les connexions entrantes.
  • Construit une classe ThreadServer pour gérer chaque client de manière indépendante.
  • Implémenté une logique de traitement basée sur des commandes texte : AUTOREGISTER, STOREMEASURE, etc.
  • Relié le serveur à un backend (MongoDB et/ou API Node.js) via la classe DataExchanger.
  • Testé les connexions avec plusieurs ESP32 en simultané et validé la robustesse du système.

Traces (captures commentées)

Extrait de code commenté 

void loop() {
    // Lire les données des capteurs
    float temperature = dht.readTemperature();
    float humidity = dht.readHumidity();
    int lightLevel = analogRead(LDRPin);
    
    // Envoyer les données au serveur
    if (client.connect(serverIP, serverPort)) {
        client.print(String(temperature) + "," + String(humidity) + "," + String(lightLevel));
        client.stop();
    }
    delay(2000); // Attendre 2 secondes avant la prochaine lecture
}

Commentaire : Ce code montre comment les données sont lues à partir des capteurs et envoyées à un serveur via une connexion réseau.

  • Testé la transmission des données en temps réel.

Contexte du projet

L’interface Frontend du projet GEST PLANTE permet à un utilisateur connecté de :

TRACE 5 — Application mobile Android (envoi d’image pour analyse)

Savoir-faire visé

Développer une application Android native permettant à l’utilisateur de prendre ou sélectionner une image, puis de l’envoyer via une socket TCP à un serveur distant pour traitement.

Contexte du projet

Dans le projet GEST PLANTE, l’application Android permet aux utilisateurs de prendre en photo une plante ou de sélectionner une image depuis leur galerie, puis de l’envoyer à un serveur d’analyse Python. Le but est de détecter visuellement des symptômes de détérioration à travers une analyse IA.
L’application a été développée en Kotlin avec Android Studio.

Illustration — Interface de l’application mobile

  • Capturer une photo en direct via la caméra
  • Choisir une image depuis la galerie
  • Envoyer l’image au serveur pour analyse automatique

Extrait de code (MainActivity.kt)

1. Initialisation et permissions

btnTakePhoto.setOnClickListener {
    if (ContextCompat.checkSelfPermission(this, Manifest.permission.CAMERA) != PackageManager.PERMISSION_GRANTED) {
        ActivityCompat.requestPermissions(this, arrayOf(Manifest.permission.CAMERA), CAMERA_PERMISSION_REQUEST_CODE)
    } else {
        launchCamera()
    }
}

Vérifie et demande dynamiquement la permission d’utiliser la caméra.

2. Capture ou sélection d’image

override fun onActivityResult(requestCode: Int, resultCode: Int, data: Intent?) {
    if (resultCode == Activity.RESULT_OK) {
        val photo: Bitmap? = data?.extras?.get("data") as? Bitmap
        selectedBitmap = photo
        appLogo.setImageBitmap(photo)
    }
}

L’image capturée ou sélectionnée est affichée dans l’application et stockée localement dans un Bitmap.

3. Envoi de l’image au serveur

val socket = Socket(serverIp, serverPort)
val outputStream = socket.getOutputStream()
bitmap.compress(Bitmap.CompressFormat.JPEG, 100, byteArrayOutputStream)
outputStream.write(byteArrayOutputStream.toByteArray())
outputStream.write("END".toByteArray()) // Fin de transmission

Le Bitmap est compressé en JPEG, envoyé via TCP au serveur Python. Un indicateur END marque la fin.

Ce que j’ai fait dans cette partie :

  • Créé l’interface utilisateur avec 3 actions : choisir, prendre, envoyer.
  • Géré les permissions système Android pour l’accès à la caméra.
  • Codé la connexion TCP et l’envoi de l’image brute au serveur.
  • Implémenté la réception de réponse du serveur et un feedback via Toast.
  • Testé l’ensemble de la chaîne : mobile → réseau → serveur.
